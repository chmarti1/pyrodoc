Unit Selection

User can click to show/hide unit selection menu. Units are set using an apply/revert modality because multiple of them can be set at once. Changing the units resets everything. Not all units work on the backend. Units only work when using POST

Substance Selection

User can select a substance via this menu. Changing the substance will completely reset all data. Ideal gases currently suffer from a backend bug on specifying T&s. Air doesn't work yet because something isn't implemented on the backed.

Property Input

This is where the user can input property data, using whatever units have been apply'd. Only two properties can be specified at one time. Adjusts visible for mp substances vs. ideal gases. Buttons of GET and POST use separate HTTP request types. GET will be removed for deploy, because it doesn't accommodate units.

Property Display Selection

User can specify which properties should be shown in the table. Expand and adjust the checkboxes as desired.

Property Table

Most interactivity provided by Datatables. Should be responsive to which properties are checked in the property selection. Lots of cleanup and display optimization is needed here, but the basic functionality exists. Prettify!

Plot

The plot responds to the points added to the table. It should display all the points that were added. It should also allow the user to click to add a point for substances (subject to the points being in range). We need better initialization to show some iso-lines for the selected substance so right now you can't really click-to-add until you've added a point manually. Hover tooltip works with extra property info. Ideal gas click-to-add doesn't work right now but that's a backend issue. Lots of prettification needed here.
